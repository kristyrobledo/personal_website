1 My first post!

Ok so here goes. First blogdown post is happening right here! H/T and thankyou to Rebecca Barter on twitter. Her simple tweet got me to bite the bullet and actually start…. No more talking about Future Kristy.

So, I thought the best first blog should be to detail this process. Maybe so I can do it again? Or better yet - to help someone else.

2 What is Blogdown?

Blogdown is an R package developed by people a lot smarter than me. It’s designed to be used within Rstudio (my safe place), so I feel like this can be done relatively easily. See these blog posts for more references and work throughs (and they are probably better written than this!)

  • Rebecca Barter
  • Allison Hall

3 What to do??

  1. Go to github and create a new repo (with a readme.md). I called my repo personal_website.
  2. Go into Rstudio http://rmarkdown.rstudio.com and create a new project>version control>git and enter in the address to your new repo. This will automatically setup from the empty repo and be linked to github. Sweet.
  3. Install blogdown with
install.packages(blogdown)

  1. Checkout the themes and suss out what you like. I copied Rebecca and went with hugo-future-imperfect. So original.

  2. Back into Rstudio and lets create the site, based on the theme example to make life easier!

new_site(theme="jpescador/hugo-future-imperfect")

Take a look at Julio’s website for more specifics on the theme.

  1. I immediately used
serve_site()

to render the site and check it out. Fun - works!!

Note: I started by using build_site() but ran into dramas. Smarter people than me found out the issue was using build_site rather than serve_site….

  1. Configured config.toml to my liking. The beauty of serve_site() is the live reloading…. love it!

  2. Next I went into Content>about>Index.md and changed this to reflect my CV.

  3. After this I decided it was time to push up to github - all using the git pane within Rstudio.

  4. Now with the shell of a website I decided it was time to try deployment….. rather than spending a lot more time on this and then failing at deployment!

  5. Head to https://www.netlify.com/ and create a free account. I created mine through github.

  6. Click on new site and create a new site from the correct github repo. Once you have deployed it, you can also change your domain - luckily there are very few Kristy Robledo’s around and I snagged https://kristyrobledo.netlify.com//.

  7. Looks like it deployed! Wow - seriously as easy as people really did write and I actually still have no idea what css is or how to code it.

  8. To create a post…. use the “Addin” bar in Rstudio and click New Post. Complete the metadata (feeds directly into your yaml) and like magic…. it just works!

AMENDMENT After trying for ages to figure out why my homepage was blank, it was pointed out to me on Slack that blogdown::serve_site() is superior to build_site(). So make sure you are using serve_site!
